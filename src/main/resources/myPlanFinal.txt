План

1.Создать классы Task, Subtask и Epic:
    а) Создать класс Task с полями: название, описание, идентификатор и статус.
    б) Создать класс Subtask, который наследуется от Task и добавляет поле для 
       хранения информации об эпике, к которому принадлежит подзадача.
    в) Создать класс Epic, который наследуется от Task и добавляет поле для хранения списка подзадач,
       входящих в эпик.

2.Создать класса TaskManager:
    а) Создать класс TaskManager для управления задачами.
    б) В классе TaskManager выбрать HashMap для хранения задач всех типов.
    в) Реализовать методы для каждого типа задач:
	Методы для задач:
	    -Получение списка всех задач.
	    -Удаление всех задач.
	    -Получение задачи по идентификатору.
	    -Создание задачи.
	    -Обновление задачи.
	    -Удаление задачи по идентификатору.
	Методы для эпиков:
	    -Получение списка всех подзадач определенного эпика.
	Методы для подзадач:
	    -Получение эпика, к которому принадлежит подзадача. (этого пункта нет в менеджере, но есть в типах задач)

3)Реализовать методы управления статусами:
    а) При обновлении статуса задачи, обновлять его вместе с полной информацией о задаче.
    б) При обновлении статуса подзадачи, проверять статусы всех подзадач в эпике и обновлять статус
       эпика.
    в) При обновлении статуса эпика, учитывать статусы всех его подзадач для определения статуса эпика.(по-моему перемудрил)

4)Тестирование:
    а) В методе main класса Main создайте экземпляр TaskManager.
    б) Создать две задачи, один эпик с двумя подзадачами, а другой эпик с одной подзадачей.
    в) Вывести списки задач, подзадач и эпиков.
    г) Изменить статусы созданных объектов и проверить, что статусы задач и эпиков обновились правильно.
    д) Удалить одну из задач и один из эпиков.
